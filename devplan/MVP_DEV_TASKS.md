# QuickTranslate MVP 开发任务拆解

## 1. 文档目标

本文件基于 `MVP_PRD.md` 输出可执行的开发任务拆解，用于指导 MVP 版本的研发推进、排期拆分与联调验收。

文档目标：

- 将 PRD 中的需求转化为工程任务
- 明确任务优先级、依赖关系与建议顺序
- 定义各阶段交付物和验收标准
- 帮助后续按模块逐步实现，而不是一次性铺开

## 2. 开发原则

- 优先打通主流程，再补齐增强能力
- 先做可运行闭环，再接入真实模型
- 播放器、字幕、转写服务、数据存储相互解耦
- 第一版优先保证稳定、简单、可验证
- 对“链接导入”“复杂编辑”“高级 AI 能力”保持克制，不进入 MVP 主线

## 3. 里程碑规划

建议拆为 4 个里程碑。

### M1. 项目骨架与主页面跑通

目标：

- App 可启动
- 首页与详情页可访问
- 可导入本地媒体并创建项目

交付结果：

- 项目基础架构完成
- 页面导航跑通
- 本地文件导入跑通

### M2. 播放器与字幕联动闭环

目标：

- 可播放音频/视频
- 可展示 mock 字幕
- 可实现字幕高亮与点击跳转
- 可实现片段循环

交付结果：

- 学习详情页主交互完成
- 精听体验基本可用

### M3. 本地转写接入

目标：

- 接入本地 ASR 模型
- 生成真实字幕
- 持久化字幕数据与状态

交付结果：

- 可对真实媒体进行本地转写
- 可复用转写结果继续学习

### M4. 稳定性与发布前收口

目标：

- 修复主流程问题
- 完成关键测试
- 补齐错误处理与状态恢复

交付结果：

- MVP 达到可内测状态

## 4. 模块拆解总览

MVP 建议拆成以下模块：

- App 基础架构模块
- 首页与项目管理模块
- 媒体导入模块
- 播放器模块
- 字幕模块
- 片段循环模块
- 本地转写模块
- 数据存储模块
- 后台任务与状态恢复模块
- 测试与质量保障模块

## 5. 详细任务拆解

## 5.1 App 基础架构模块

### 目标

建立后续开发的基础结构，避免功能逐步堆积后难以维护。

### 任务列表

- 确定项目整体架构
  - 建议：单 app 模块起步，按 feature 分包
- 确定 UI 技术方案
  - 建议：Jetpack Compose
- 确定状态管理方式
  - 建议：ViewModel + StateFlow
- 确定导航方案
  - 建议：首页 -> 详情页
- 接入依赖管理
  - ExoPlayer / Media3
  - Room
  - 协程
  - 文件选择器相关依赖
- 建立基础目录结构
  - `feature/home`
  - `feature/session`
  - `feature/transcription`
  - `data`
  - `domain`
  - `player`

### 输出物

- 可运行项目骨架
- 基础依赖已接入
- 导航结构已建立

### 优先级

最高

### 依赖关系

无

## 5.2 首页与项目管理模块

### 目标

支持用户进入 App 后快速开始新学习或继续已有项目。

### 任务列表

- 设计首页 UI
- 实现首页空状态
- 实现最近项目列表
- 实现项目卡片展示
  - 文件名
  - 类型
  - 最近学习时间
  - 字幕状态
- 实现点击项目进入详情页
- 实现删除项目确认交互

### 数据相关任务

- 定义 `Project` 数据模型
- 建立项目列表查询逻辑
- 建立删除项目逻辑

### 输出物

- 首页 UI 完成
- 最近项目列表可正常展示
- 项目删除可用

### 优先级

高

### 依赖关系

- 依赖基础架构模块
- 部分依赖数据存储模块

## 5.3 媒体导入模块

### 目标

支持用户通过系统文件选择器导入本地音频或视频，并生成学习项目。

### 任务列表

- 接入系统文件选择器
- 限制可选文件类型
  - 音频
  - 视频
- 获取导入文件的 URI
- 读取文件元信息
  - 文件名
  - MIME type
  - 时长
- 校验文件是否可用
- 创建项目记录
- 导入完成后跳转学习详情页

### 异常处理任务

- 文件不可访问提示
- 不支持格式提示
- 元数据读取失败提示

### 输出物

- 本地文件导入可用
- 导入后可创建项目并进入详情页

### 优先级

最高

### 依赖关系

- 依赖基础架构模块
- 依赖项目存储能力

## 5.4 播放器模块

### 目标

实现音频/视频播放与播放状态分发，为字幕联动和循环播放提供基础能力。

### 任务列表

- 接入 Media3 / ExoPlayer
- 建立播放器控制封装层
- 支持加载本地 URI 媒体
- 支持音频播放
- 支持视频播放
- 支持基础控制
  - 播放
  - 暂停
  - seek
  - 快进
  - 快退
- 暴露播放器状态
  - 当前播放位置
  - 总时长
  - 是否正在播放
  - 加载状态
- 详情页接入播放器 UI

### 技术注意点

- 播放器生命周期与页面生命周期解耦
- 退出页面时保存最近播放位置
- 避免高频状态更新导致 UI 卡顿

### 输出物

- 本地音频/视频可播放
- 详情页可显示播放控制

### 优先级

最高

### 依赖关系

- 依赖媒体导入模块

## 5.5 字幕模块

### 目标

支持字幕数据展示、当前句高亮、字幕点击跳转与选区管理。

### 任务列表

- 定义 `SubtitleSegment` 数据模型
- 实现字幕列表 UI
- 实现当前字幕高亮逻辑
- 实现根据当前播放时间匹配字幕片段
- 实现点击字幕跳转
- 实现列表自动滚动到当前句
- 实现字幕选中态
- 实现多条连续字幕选择逻辑

### 交互任务

- 当前句高亮样式设计
- 选中句样式设计
- 字幕为空状态设计
- 字幕生成中状态展示

### 输出物

- 字幕列表展示可用
- 字幕与播放进度联动可用
- 点击字幕跳转可用

### 优先级

最高

### 依赖关系

- 依赖播放器模块
- 依赖数据存储模块

## 5.6 片段循环模块

### 目标

支持对单句或多句字幕构成的区间做循环播放。

### 任务列表

- 定义 `LoopConfig` 数据模型
- 实现字幕选区到循环区间的映射
- 实现循环设置面板
- 支持循环次数选项
  - 1 次
  - 3 次
  - 5 次
  - 无限
- 实现循环状态机
  - 开始循环
  - 到达终点自动 seek
  - 次数累计
  - 达到上限停止
- 实现手动停止循环
- 在 UI 中展示循环中状态

### 技术注意点

- 处理 seek 回起点时的时间误差
- 避免播放结束与循环判断冲突
- 保证暂停、切后台、恢复播放时逻辑一致

### 输出物

- 单句循环可用
- 多句循环可用
- 次数限制可用

### 优先级

高

### 依赖关系

- 依赖播放器模块
- 依赖字幕模块

## 5.7 本地转写模块

### 目标

把已导入媒体转换成带时间戳的字幕片段。

### 建议分两步实施

#### 第一步：Mock 转写

- 构建统一转写接口
- 输入：媒体 URI
- 输出：字幕片段列表
- 使用 mock 数据先驱动 UI 与联动逻辑

#### 第二步：真实模型接入

- 确定模型方案
  - 优先候选：`whisper.cpp`
  - 可选候选：`sherpa-onnx`
- 实现媒体音轨读取与预处理
- 将音频数据交给本地模型推理
- 解析输出为字幕片段
- 写入本地数据库

### 状态任务

- 定义转写状态
  - idle
  - queued
  - processing
  - success
  - failed
- 详情页展示状态反馈
- 支持重试

### 风险点

- 模型体积与初始化耗时
- 音频预处理格式适配
- 设备性能差异
- 长音频下的耗时与内存占用

### 输出物

- Mock 转写能力
- 真实本地转写能力
- 转写结果可落库并回显

### 优先级

高

### 依赖关系

- 依赖媒体导入模块
- 依赖数据存储模块
- 依赖后台任务模块

## 5.8 数据存储模块

### 目标

持久化项目、字幕和播放状态，支持应用重启后恢复。

### 任务列表

- 选定本地存储方案
  - 建议：Room
- 设计数据表
  - Project
  - SubtitleSegment
  - LoopConfig
- 实现 DAO
- 实现 Repository
- 保存最近播放位置
- 保存转写状态
- 保存字幕结果
- 保存最近循环配置

### 输出物

- 项目、字幕、循环配置均可持久化
- 重启应用后可恢复数据

### 优先级

最高

### 依赖关系

- 依赖基础架构模块

## 5.9 后台任务与状态恢复模块

### 目标

让转写等长耗时任务在后台稳定执行，并在页面重进时可恢复状态。

### 任务列表

- 建立长任务执行机制
  - 可选：WorkManager 或前台服务
- 实现转写任务调度
- 实现任务状态回传
- 实现失败状态回写
- 实现详情页重新进入后的状态恢复
- 处理中状态的 UI 持续反馈

### 技术注意点

- 长任务不能阻塞主线程
- 任务中断后需要可识别状态
- 页面销毁后状态不能丢

### 输出物

- 转写任务可在后台稳定运行
- 页面重进后可恢复转写状态

### 优先级

高

### 依赖关系

- 依赖本地转写模块
- 依赖数据存储模块

## 5.10 测试与质量保障模块

### 目标

保证 MVP 主流程稳定可演示、可内测。

### 任务列表

- 单元测试
  - 字幕时间命中逻辑
  - 循环次数计算逻辑
  - 选区时间范围计算逻辑
- 集成测试
  - 文件导入到项目创建链路
  - 项目恢复链路
- UI 测试
  - 首页导入流程
  - 详情页播放交互
  - 字幕点击跳转
- 手工测试清单
  - 音频文件
  - 视频文件
  - 短音频
  - 长音频
  - 转写失败重试
  - 应用切后台恢复

### 输出物

- 核心逻辑测试
- 主流程测试清单
- 已知问题列表

### 优先级

高

### 依赖关系

- 依赖所有功能模块

## 6. 推荐开发顺序

建议按以下顺序推进：

1. 基础架构模块
2. 数据存储模块
3. 首页与项目管理模块
4. 媒体导入模块
5. 播放器模块
6. 字幕模块
7. 片段循环模块
8. Mock 转写模块
9. 真实本地转写模块
10. 后台任务与状态恢复模块
11. 测试与质量保障模块

说明：

- 先把“导入 -> 播放 -> mock 字幕 -> 循环”做通，最快能看到产品形态
- 真实转写放在中后段接入，避免模型问题阻塞 UI 与交互开发

## 7. 建议任务分期

## Phase 1：可交互 Demo

目标：

- 首页
- 文件导入
- 详情页播放器
- mock 字幕
- 字幕联动
- 片段循环

适合验收的问题：

- 用户是否理解这个产品在干什么
- 当前交互是否顺手
- 字幕与循环训练体验是否成立

## Phase 2：可用 MVP

目标：

- 接入真实本地转写
- 状态持久化
- 历史项目恢复
- 错误处理

适合验收的问题：

- 本地转写是否足够实用
- 端侧性能是否可接受
- 长音频场景是否稳定

## Phase 3：内测优化版

目标：

- 修复稳定性问题
- 优化 UI 细节
- 补齐必要测试
- 调整模型和默认参数

适合验收的问题：

- 是否具备发给少量用户内测的质量

## 8. 每阶段验收清单

## M1 验收

- App 可运行
- 首页可展示空状态
- 可从首页进入导入流程
- 导入成功后可创建项目并进入详情页

## M2 验收

- 音频可播放
- 视频可播放
- mock 字幕可展示
- 当前字幕可高亮
- 点击字幕可跳转
- 单句与多句循环可用

## M3 验收

- 可触发真实本地转写
- 可看到转写进度状态
- 转写成功后字幕可落库
- 再次打开项目时字幕仍存在

## M4 验收

- 后台恢复可用
- 失败重试可用
- 主流程无明显阻塞型 Bug
- 可完成一轮端到端学习流程

## 9. 风险清单

- 本地模型集成耗时可能明显高于 UI 开发
- 视频文件的音轨提取与格式兼容可能出现额外工作量
- 低性能设备上转写速度可能不理想
- 长音频的转写与字幕列表渲染可能暴露性能问题
- 循环播放边界处理可能出现 seek 抖动和重复次数误差

## 10. 建议的最小实施版本

如果要以最短时间做出第一个可演示版本，建议只做以下任务：

- 首页
- 本地文件导入
- 项目持久化
- 音频播放
- mock 字幕展示
- 当前句高亮
- 点击字幕跳转
- 单句循环播放

这一版完成后，就已经可以验证核心价值。随后再增加：

- 视频播放
- 多句循环
- 真实本地转写
- 后台任务与恢复
